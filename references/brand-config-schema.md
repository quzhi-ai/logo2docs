# brand-config.json Schema Reference

This document describes every field in brand-config.json. The config is generated by `scripts/brand_setup.py` and consumed by all document generation workflows.

## Structure

```json
{
  "meta": { ... },
  "logo": { ... },
  "colors": { ... },
  "typography": { ... },
  "layout": { ... },
  "html": { ... },
  "tables": { ... },
  "charts": { ... }
}
```

## meta

| Field | Type | Description |
|-------|------|-------------|
| company_name | string | Company name, used in headers/footers |
| industry | string | Industry category (logistics, tech, finance, retail, etc.) |
| style | string | One of: professional, modern, bold, elegant |
| generated_at | ISO string | When the config was generated |
| generator | string | Always "brand-office-suite" |

## logo

| Field | Type | Description |
|-------|------|-------------|
| color_path | string | Relative path to color logo file |
| white_path | string | Relative path to white/inverse logo |
| aspect_ratio | number | Width / height ratio (e.g., 2.113 means 2.113x wider than tall) |
| has_white_version | boolean | Whether a white version is available |
| base64_color | string | Data URI of color logo for HTML embedding |
| base64_white | string | Data URI of white logo for HTML embedding |
| safe_zone_ratio | number | Minimum clearance around logo as ratio of logo height (default 0.5) |
| primary_luminance | number | 0-1, luminance of primary brand color (helps decide light/dark logo) |

**Logo usage rule**: When placing the logo, check the background's luminance. If < 0.5 (dark), use `base64_white`. If >= 0.5 (light), use `base64_color`. If no white version exists and background is dark, add a light container behind the color logo.

## colors

### Brand colors

| Field | Example | Usage |
|-------|---------|-------|
| primary | #045e81 | Headers, titles, emphasis, primary buttons |
| primary_dark | #03495f | Hover states, pressed buttons, dark accents |
| primary_light | #e8f4f8 | Subtle backgrounds, tints, callout backgrounds |
| secondary | #15a04a | Accents, tags, status positive, chart highlight |
| secondary_dark | #0e7a38 | Secondary hover/press states |
| secondary_light | #e8f8ee | Secondary tint backgrounds |

### colors.text

| Field | Example | Usage |
|-------|---------|-------|
| primary | #2d3748 | Main body text (never use brand colors for body text) |
| secondary | #4a5568 | Less prominent text, descriptions |
| muted | #8896a6 | Timestamps, metadata, footnotes |
| on_primary | #ffffff | Text placed on primary-colored backgrounds |
| on_secondary | #ffffff | Text placed on secondary-colored backgrounds |

### colors.background

| Field | Example | Usage |
|-------|---------|-------|
| page | #f8f9fa | Page/document background (slightly tinted toward brand) |
| card | #ffffff | Card/container backgrounds |
| subtle | #e8f4f8 | Subtle emphasis areas (same as primary_light) |

### colors.functional

| Field | Example | Usage |
|-------|---------|-------|
| success | #16a34a | Positive status, completion, above-target |
| warning | #d97706 | Caution, approaching limit, needs attention |
| error | #dc2626 | Negative status, failure, below-target |
| info | (=primary) | Informational callouts, neutral status |

### colors.chart_series

Array of 5-7 hex colors for data visualization. First two are primary and secondary; remaining are generated to be visually harmonious and distinguishable.

## typography

### heading

| Field | Example | Description |
|-------|---------|-------------|
| family_web | Manrope, sans-serif | Font family for HTML/PDF outputs |
| family_office | Calibri | Font family for Office (xlsx/docx/pptx) |
| weight | 700 | Font weight for headings |

### body

Same structure as heading, typically weight 400.

### mono

| Field | Example | Description |
|-------|---------|-------------|
| family | Consolas, monospace | For code, data values, technical content |

## layout

| Field | Example | Description |
|-------|---------|-------------|
| max_content_width | 1100px | Maximum width for HTML content |
| page_margin_cm | 2.5 | Document margins in centimeters (Word/PDF) |
| section_gap | 32px | Space between major sections |
| paragraph_gap | 16px | Space between paragraphs |
| border_radius | 6px | Default corner radius for cards/containers |
| shadow | 0 1px 3px rgba(...) | Default box shadow for elevated elements |

## html.css_variables

A flat dictionary mapping CSS custom property names to values. Copy these into a `:root {}` block at the top of any generated HTML for consistent theming.

## tables

| Field | Description |
|-------|-------------|
| header_bg | Background color for table headers (= primary) |
| header_text | Text color in headers (= on_primary) |
| row_alt_bg | Alternating row background |
| row_bg | Default row background |
| border_color | Horizontal divider color |
| border_style | "horizontal_only" — no vertical borders, no full grid |

## charts

| Field | Description |
|-------|-------------|
| max_pie_slices | Max 5 slices; 6+ becomes "Other" |
| line_width_px | Line thickness in line charts |
| data_point_radius_px | Data point circle radius |
| max_lines_per_chart | Max 4 lines per chart for readability |
| series_colors | Same as colors.chart_series |
